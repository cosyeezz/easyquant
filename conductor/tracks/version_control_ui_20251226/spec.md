# Spec: 版本管理界面与画布集成

## 目标
在前端实现完整的版本管理交互，让用户能够查看节点的历史版本，并在 SNAPSHOT 和 RELEASE 之间切换，同时确保画布能够引用正确版本的节点定义。

## 核心需求
1. **版本历史视图:** 在节点管理页面添加一个侧边栏或模态框，列出所有历史版本及变更日志。
2. **版本切换交互:** 支持一键回滚到旧版本或将旧版本设为当前草稿。
3. **节点选择器集成:** 节点选择器（Node Selector）应支持展开版本列表，允许用户选择特定的 RELEASE 版本添加到画布。
4. **状态同步:** 确保画布上的节点实例能够正确感知其所属的版本状态。
