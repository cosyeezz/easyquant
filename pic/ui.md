角色设定：
你现在是一名世界顶级的资深 UI/UX 设计师，擅长 FinTech（金融科技）领域和高性能 SaaS 后台管理系统的设计。你的设计风格对标 Linear.app, Vercel 以及 Stripe Dashboard，追求极简、精密、科技感与极致的用户体验。

项目背景：
项目名为“EasyQuant”，是一个量化交易监控系统。目前系统的 UI 虽然功能完备，但视觉表现较为传统（类似于通用的 B 端管理后台模板），缺乏专业量化工具应有的“精密感”、“高级感”和“信息密度控制”。

重构目标：
请基于现代视觉趋势，为 EasyQuant 打造一套全新的视觉语言体系。设计需要传达出：可靠、实时、高效、专业。

请从以下维度提供详细的重构方案：

1. 视觉风格与哲学 (Visual Philosophy)：

去模板化： 抛弃沉重的边框和阴影，转向“Linear 风格”——利用 1px 的精致边框、微妙的灰度变化和大量的留白（Negative Space）来区分层级。
模块化容器： 采用“Bento Box（便当盒）”或模块化卡片布局，让每一个数据模块看起来像是一个独立的精密仪器。
2. 色彩系统与设计令牌 (Design Tokens)：

主色调升级： 建议将目前的普通蓝色重构为更具深度的“深空蓝”或“高饱和度的电动蓝”（例如 #0061FF 或 #2563EB）。
中性色阶： 定义一套多层的灰色系统（从纯白背景到浅灰容器再到深灰文字），确保背景、卡片和边框之间有极细微但可感知的差异。
语义色优化： 针对“在线/成功”的绿色和“错误/预警”的红色，提供更具高级感的低饱和度配比（如 Tailwind 的 Emerald-500 和 Rose-500 系列）。
3. 核心组件精细化设计：

顶部状态栏 (System Stats)： 重新设计 CPU、内存和在线状态。不要使用大面积色块，尝试使用“呼吸灯状态点 + 细体数值 + 微型趋势图 (Sparkline)”的组合。
数据表 (Data Table) 层次感：
文字层级： 标题使用粗体加黑，副标题/描述文字使用小号中性灰。
等宽字体： 对于物理名、数值等数据，强制要求使用等宽字体（Monospace），以确保纵向对齐的严整性。
标签 (Tags) 进化： 将厚重的标签改为“浅色底+深色字”或“无底色+圆点指示器”的极简样式。
搜索与筛选区： 设计得更轻量化，例如采用内嵌式图标搜索框，并在非聚焦状态下减弱边框存在感。
4. 字体排版 (Typography)：

推荐一套适合金融后台的字体栈（如 Inter, Geist, 或 SF Pro）。
说明如何通过字母间距 (Letter-spacing) 和行高 (Line-height) 来提升大量数据排版时的可读性。
